About Sileo Native Depiction Generator

Sileo Native Depiction Generator is a script that allows you to generate/create native depictions for the Sileo application by using a premade template. Sileo Native Depiction Generator is writen in Python 3.x.

Using Sileo Native Depiction Generator

Sileo Native Depiction Generator is a simple and easy to use script to generate/create native depictions for the Sileo application.

What Do I Need?

- Python 3.x.
- A computer or pretty much any OS that can run Python, this includes Windows, Mac, Linux, etc...

Creating Your First Depiction File

First you will download Sileo Native Depiction Generator then open/run generator.py with Python and follow the questions on screen. Once completed the script will save your depiction file to the set path thats in config.py (default: depiction/user/templates).

How To Use Custom Templates

You can use custom templates with Sileo Native Depiction Generator you just have to manually add them and set it up.

Placeholders

Sileo Native Depiction Generator uses a template with placeholder text inside then replaces the placeholder text with the users input when the script is ran. To set up these placeholders open your depiction file in a text editor and start replacing the text with the placeholder text, for example if you have a place where your description will be for that package replace it with something like "Package Description". You can use whatever text you want just remember them because you will need to update the script with them later. Some examples of placeholder text are:
Header Image URL Package Name Package Description Package Known Issues First Released Version Number First Release Date Latest Version Number Package Price Package Developer Support URL

Your template may not use all of thoses but it might uses some.

Adding Template To Script

Once you have replaced everything with your placeholder text add the template to the folder where all your other templates are so in depictions/templates unless you changed it in config.py.
1. Open generator.py with a text editor.
2. Find a if or elif statement so something like if (template == "Template X" or "X"): and copy that code all the way to the next if or elif statement.
3. Now go to the bottom of the script and above the last else paste that whole code and change if to elif if needed. Now go back to the top of the code you just pasted.
4. Now once at the top find if (template == "Template X" or "X") now replace Template X with the name to find the template when using the script or you can replace X with a number. Note: Make sure you don't use duplicate names this can cause problems.
5. Now find TemplateFileName and change whats insinde the "" (after the = sign) to your template's file name thats in depictions/templates.
6. Start calling or removing functions that contain the questions to fit your template. Please don't remove PackageIDQuestion() as it is required unless changed.
7. Now go down till you find replacements and replace the placeholder text to the text you are looking for.
Example:
'Package Name':packagename
What this is doing is telling it to look for the placeholder text which in this case it's Package Name and replace it with what the user enters.
8. Now right under that you should see removeplaceholdertext and what this does is tell the script to change the fields that hasn't been updated/change by the script to nothing so you can update this or in some cases leave it alone.
9. Now go down till you see something like full_data['tabs'][X]['views'][X]['screenshots'] (X will be replaced with some number). Now go back to your template and find where screenshots would be so in template2.json it's at line 32 so I would replace that with full_data['tabs'][0]['views'][4]['screenshots']. What this is saying that screenshots can be found inside tabs and inside that views then down 4 then find screenshots. Once you find that and replace it make sure also replace something like this full_data['tabs'][X]['views'][X]['screenshots'] = screen_shots (X will be replaced with numbers) with the same thing above now do the same for the else statement.
10. Now run the script and test your template.

Functions

PackageIDQuestion() - This function is used to name the depiction file after it's created.
HeaderImageQuestion() - This function is to ask the user what the link is to a header image they want to use in the depiction file.
PackageNameQuestion() - This function is to ask the user what the name is for their package.
PackageDescripionQuestion() - This function is to ask the user what their package is about.
PackageUseScreenshots() - This function is to ask the user if they want to use screensots for their package.
KnownIssuesQuestion() - This function is to ask the user what are the known issues with their packages.
FirstVersionQuestion() - This function is to ask the user what was the first version number for their package was.
FirstReleaseDateQuestion() - This function is to ask the user the first released date for their package.
LatestVersionQuestion() - This function is to ask the user what is their latest version for their package.
LatestReleasedDateQuestion() - This function is to ask the user what is the latest released data for their package.
PriceQuestion() - This function is used to ask the user the price for their package.
DeveloperQuestion() - This function is to ask the user who is the developer for that package.
SupportURLQuestion() - This function is to ask the user what is a url someone could use to contact the developer.

Frequently Asked Questions

Can I Use Sileo Native Depiction Generator To Edit A Template?

No Sileo Native Depiction Generator does not have a system to let you edit templates with the script itself.

Why Can't I Add Changelog/Version History?

Sileo Native Depiction Generator does not have support yet for adding changelog/version history to a depiction file but once it's created you can edit it with a text editor and add it yourself.
